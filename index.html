// FORMATS DE FICHIERS

Guide de référence pour préparer vos données avant chargement dans l'application.

📂 Bathymétrie — 3 formats acceptés
Format 1 · CSV / XYZ recommandé

Tableau de valeurs séparées par virgule, point-virgule ou tabulation. Chaque ligne = un point de mesure.

Colonnes requises (dans n'importe quel ordre) :
latitudedécimal · ex : 34.85
longitudedécimal · ex : 138.55
profondeurvaleur positive en mètres
Règles :
  • 1 ligne d'en-tête optionnelle (ignorée automatiquement si elle contient des lettres)
  • Profondeur en valeur positive — ex : 2400 et non -2400
  • Séparateur accepté : virgule , · point-virgule ; · tabulation \t · espace
  • Taille max recommandée : 500 000 points (l'app sous-échantillonne automatiquement si dépassé)
  • Encodage UTF-8 ou ASCII
# exemple valide lat,lon,depth 34.85,138.55,1200 34.86,138.56,1350 34.84,138.54,980
Format 2 · ASC (Esri ASCII Grid) GEBCO

Format grille raster exporté par GEBCO, QGIS ou ArcGIS. Extension .asc. Contient un en-tête de métadonnées suivi des données.

Structure attendue :
ncols 3601 nrows 3001 xllcorner 135.0 yllcorner 32.0 cellsize 0.01 nodata_value -9999 0 0 -200 -450 -800 ...
Règles :
  • Mots-clés d'en-tête : ncols, nrows, xllcorner / xllcenter, yllcorner / yllcenter, cellsize
  • nodata_value optionnel (défaut : -9999)
  • Valeurs de profondeur : négatives pour l'océan — l'app les convertit automatiquement en positif
  • Origine xll/yll = coin bas-gauche en degrés décimaux WGS84
  • Sur GEBCO Download → choisir le format : Esri ASCII Raster (.asc)
🌊 Courants océaniques — Format CSV
Format CSV u/v requis

Chaque ligne représente un point de grille avec les deux composantes de vitesse horizontale.

Colonnes requises :
latlatitude décimale
lonlongitude décimale
uvitesse Est–Ouest (m/s)
vvitesse Nord–Sud (m/s)
Règles :
  • Unités u et v : mètres par seconde (m/s) — l'app affiche en cm/s automatiquement
  • 1 ou 2 lignes d'en-tête tolérées
  • Les valeurs NaN, vides ou nulles sont ignorées
  • Grille régulière recommandée mais non obligatoire
  • Longitude en −180/180 ou 0/360, les deux sont acceptés
# exemple valide lat,lon,water_u,water_v 34.0,138.0,0.12,-0.05 34.0,138.08,0.09,-0.03 34.08,138.0,0.15,-0.07
⚙ Conversion NetCDF HYCOM → CSV (Python) :
import xarray as xr ds = xr.open_dataset("hycom.nc") df = ds[["water_u","water_v"]].isel(time=0, depth=0) \ .to_dataframe().reset_index() df[["lat","lon","water_u","water_v"]] \ .dropna().to_csv("courants.csv", index=False)
🪨 Lithologie / Sédimentologie — Format CSV
Format CSV sédiments requis

Chaque ligne décrit un point de grille avec son type de sédiment codé numériquement.

Colonnes requises :
latlatitude décimale
lonlongitude décimale
soil_identier 0–254
labelnom du type (texte libre)
Règles :
  • soil_id : entier entre 0 et 254 — la valeur 255 est réservée "non défini" et ignorée
  • Tous les points avec le même soil_id reçoivent automatiquement la même couleur
  • Champ label libre : ex. argile pélagique, sable fin, basalte
  • La bathymétrie doit être chargée d'abord — nécessaire pour masquer les zones terrestres
  • Taille max recommandée : 300 000 points
# exemple valide lat,lon,soil_id,label 34.0,138.0,3,Argile pélagique 34.0,138.08,5,Sable fin 34.08,138.0,3,Argile pélagique
⚙ Conversion EarthByte seabed_lithology_v1.nc → CSV (Python) :
import xarray as xr, pandas as pd, numpy as np ds = xr.open_dataset("seabed_lithology_v1.nc") # Voir les variables disponibles print(list(ds.data_vars)) # ex: ['lid'] var = "lid" # ou le nom trouvé ci-dessus da = ds[var].squeeze() lats = da.coords["latitude"].values lons = da.coords["longitude"].values data = da.values # Sous-échantillonnage ×4 pour réduire la taille stride = 4 rows = [] for i in range(0, len(lats), stride): for j in range(0, len(lons), stride): v = data[i, j] if np.isfinite(v) and v >= 0: rows.append((lats[i], lons[j], int(v), str(int(v)))) df = pd.DataFrame(rows, columns=["lat","lon","soil_id","label"]) df.to_csv("lithologie.csv", index=False) print(f"{len(df)} points exportés")
